- MySQL
	In Myqls Left join, cross join e inner join are equivalents 
		http://i.stack.imgur.com/1UKp7.png
		- Left Join
			- Gives a extra consideration to the table that is on the left
			- Return all rows from the left table, with the matching rows in the right table 
			- Every items in the left table will show up in result even if there isn't match with other table.
			http://www.w3schools.com/sql/img_leftjoin.gif
			ex:
			Return all clients and its "pedidos"
			SELECT cliente.nome_cliente, pedido.num_pedido, pedido.prazo_entrega FROM cliente
					LEFT JOIN pedido
					ON cliente.nome_cliente = pedido.codigo_cliente;
		- INNER JOIN
			- Return the intersection between two tables
			http://www.w3schools.com/sql/img_innerjoin.gif
			ex: Return all clients and its "pedidos"
			SELECT cliente.nome_cliente, pedido.num_pedido, pedido.prazo_entrega FROM cliente
					INNER  JOIN pedido
					ON cliente.nome_cliente = pedido.codigo_cliente;

	- Low Coupling
		Isolate peaces of code that can be reuse.  Have a good interface between objects to alow them to communicate only with small number of connections.  The classs do not depend each other to do a task.
	- High Cohesion
		The object do exactly what it is projected to do.  Closely related to low coupling.  Building small classes that do completely their tasks, the classes are cohese.
		A Low coupling class need to be constructed in small portions doing some definite tasks.
